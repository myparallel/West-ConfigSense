## 1. 项目概述

### 1.1 项目背景

#### 1.1.1 行业趋势：从"植物互联网"到智慧农业

> *"为确保可持续的未来并对抗食物短缺，我们必须提高农业生产力，增强气候韧性并优化资源使用。在农业中，尚未充分利用的密集无线传感器网络可以在与智能决策和控制系统连接时增加产量并支持韧性生产。"*
> — Nature, 2025

根据Nature期刊（2025）发布的"农业传感器：构建植物互联网时代"前瞻性研究，全球农业正从传统经验驱动转向**数据驱动的精准农业时代**。该研究提出了**植物互联网（Internet of Plants, IoP）**的概念——通过部署密集的无线传感器网络，作为农民的"眼睛、鼻子和触觉"，实时感知环境参数与植物生理指标，并与智能决策和控制系统连接，实现：

- **高空间分辨率感知**：多种环境参数的密集采集（土壤pH/湿度/温度/NPK、气象温湿度/CO₂/辐射/风速等）
- **植物生理监测**：茎流、茎粗、病虫害、叶色等作物健康指标的实时检测
- **智能决策与控制**：传感器数据联动灌溉、施肥、环控等执行系统，实现自主化作物管理
- **成本降低与规模化部署**：传感器成本持续下降（气象传感器约1欧元/个，气体传感器约10欧元/个），使大规模部署成为可能

这一趋势与WEST公司的战略方向高度契合：**传感器硬件是基础，数据汇聚是核心，智能应用是价值，而连接这一切的软件平台是关键缺失环节。**

![IoP架构示意图](ConfigSense_IoP_architecture.png)
*图：植物互联网（IoP）架构 — 传感器网络感知环境与植物生理参数，经通信网关上传云端，实现精准农业管理*

#### 1.1.2 公司背景与战略转向

WEST公司（海南世电科技有限公司）是一家专注于工业物联网的方案和设备供应商，秉承集团近40年商贸资源，在物联网产业链方向持续研发，产品覆盖感知层（传感器/变送器）、传输层（网关/通信模块）、应用层（平台软件）的完整物联网三层架构。

公司在上海、苏州、成都、台北四地设有研发中心，拥有13名研发工程师，年推120款新产品，已获得ATEX防爆认证和CE认证。产品通信能力覆盖LoRa、4G/Cat1、Zigbee、UWB、NB-IoT、WiFi、蓝牙、RS485等多种协议，在阿里巴巴国际站12个产品分类中，有8个直接与农业相关。

2026年起，外贸团队聚焦农林牧渔行业，进行智慧种植、智慧牧业、智慧水产养殖等行业的物联网硬件产品推广和销售。但目前公司缺乏一套通用的应用平台WEB软件和手机APP软件，无法为海外客户提供完整的"硬件+软件"解决方案，制约了产品竞争力和市场拓展。

#### 1.1.3 核心痛点

| 痛点 | 说明 |
|------|------|
| **硬件无"魂"** | 传感器只输出数据，缺乏面向行业的智能应用软件，客户无法直接使用 |
| **行业门槛高** | 不同行业（种植/牧业/水产）需求差异大，逐一开发成本高、周期长 |
| **数据孤岛** | 传感器数据散落在不同平台，无法汇聚、分析和联动控制 |
| **海外合规缺失** | 现有平台无多语言、无GDPR合规、无国际化地图，无法支撑海外销售 |
| **溯源能力弱** | 缺乏农产品数字护照功能，无法满足海外市场对食品安全溯源的需求 |

### 1.2 项目目标

开发一套**面向农林牧渔全行业兼容的物联网应用平台**（WEB端 + 手机APP），以数据汇聚池为核心，以行业应用逻辑和专业呈现为特色，以带动自有硬件产品销售和业务拓展为最终目标。

## 2. 平台名称深度阐述 — 灵配智控 / ConfigSense

> *"灵，是感知万物的敏锐；配，是因需而变的智慧；智，是数据驱动的洞察；控，是精准执行的可靠。灵配智控——让每一个行业，都能找到属于自己的智慧答案。"*

### 2.1 中文名称释义："灵配智控"

#### "灵" — 智慧感知，灵活响应

取自智慧、灵敏、灵巧之意。寓意平台具备高度智能化的感知能力，能够灵活响应不同场景、不同行业的复杂需求。此字不仅体现了"智慧感知"的技术内核，也暗含了平台**"用则灵、变则通"**的核心理念。

#### "配" — 按需配置，随需而动

直指"配置"这一核心功能机制。正如平台定位所言，通过指标库的属性配置与归集，即可将平台适配至农、林、牧、渔乃至家庭绿地、种植园、苗圃等任意行业。"配"字精准传达了**"按需配置、随需而动"**的平台灵魂。

#### "智" — 智慧决策，数据驱动

代表智慧决策与自动控制。平台集成了智慧感知、无线通讯与自动控制三大核心技术，通过数据驱动的智能分析，实现从"感知"到"执行"的闭环。

#### "控" — 自动控制，精准执行

直接对应"自动控制"能力。无论是灌溉控制、环境调节还是设备调度，"控"字概括了平台的执行层能力。

### 2.2 英文名称释义："ConfigSense"

#### Config — 配置

是Configuration（配置）的行业通用缩写，在国际上具有高度识别度和专业性。在软件工程与物联网领域，"Config"几乎是"灵活配置"的标准表达。例如腾讯云的"配置审计（Config）"产品即采用此命名逻辑。

#### Sense — 感知

代表感知（Sensing），直接呼应"WEST智慧感知"的核心技术层。在开源情报平台"Sense"中，同样采用此词表达"敏锐感知环境变化"的含义。

#### 组合意义

ConfigSense清晰传达了**"通过配置实现智慧感知"**的平台能力，即系统可根据用户对不同行业的指标库属性进行灵活配置，从而精准适配目标场景。

### 2.3 与"WEST"技术底盘的融合

#### 2.3.1 WEST三大核心技术支柱

"WEST"在本平台中代表三大核心技术支柱：

| 字母 | 技术维度 | 详细含义 |
|------|----------|----------|
| **W** | 无线通讯（Wireless） | 实现设备间的高效数据互联 |
| **E** | 边缘计算/环境感知（Edge/Environment） | 在靠近数据源的位置进行实时处理与感知 |
| **S** | 智慧感知（Smart Sensing） | 通过传感器网络精准采集环境数据 |
| **T** | 自动控制（Automatic Control） | 基于数据分析结果执行自动化作业指令 |

"灵配智控"中的**"智控"**直接对应**"T（控制）"**，而**"灵"**则涵盖 **W（无线通讯）、E（边缘/环境感知） 与 S（智慧感知）** 的灵巧融合。

#### 2.3.2 "配置"机制如何服务于WEST技术栈

在平台架构中，"灵配"机制具体体现在以下环节：

- **行业指标库的灵活归集**：用户可根据当前应用行业（如农业、林业、牧业、渔业或家庭绿地），在系统后台选择或自定义相应的指标库参数（如土壤湿度、光照强度、气象数据等），系统自动将"感知层"能力与行业需求匹配。
- **通讯协议的可配置化**：对于不同场景下的无线通讯需求（如LoRa、NB-IoT、Zigbee等），平台支持协议配置切换，无需更换底层硬件。
- **控制策略的动态调整**：用户可根据季节、作物类型或养殖周期，灵活设定自动控制的触发条件与执行逻辑，实现"一平台多用"的通用性目标。

#### 2.6.1 差异化竞争优势

## 3. 平台定位与核心设计理念

### 2.1 平台定位

面向最终用户（农场主、养殖户等）的 **SaaS 物联网应用平台**，同时支持私有化部署。

### 2.2 核心设计理念

> **"数据汇聚池 + 行业应用引擎"**

平台不是设备管理平台，而是以数据为根基的应用平台：

**ConfigSense 平台分层架构：**

| 层级 | 模块 | 说明 |
|------|------|------|
| **第四层：行业应用引擎** | 智慧种植（专业UI） | 种植行业专属界面、功能与控制逻辑 |
| | 智慧牧业（专业UI） | 牧业行业专属界面、功能与控制逻辑 |
| | 水产养殖（专业UI） | 水产行业专属界面、功能与控制逻辑 |
| | 泳池/草坪（专业UI） | 家庭场景专属界面、功能与控制逻辑 |
| **第三层：AI引导与场景切换** | AI行业引导 & 场景切换 | 基于指标-行业映射的智能推荐，行业切换时自动适配UI和功能 |
| **第二层：指标数据池（核心）** | 指标数据池 | 指标定义 · 行业映射 · 数据归类 — 平台的核心灵魂 |
| **第一层：统一数据接入** | 统一数据接入层 | 多协议 · 多源 · 最大兼容 |
| | WES-AIOT 数据传入 | WEST自有IoT平台数据通道 |
| | 第三方API/SDK接入 | 外部系统标准接口推送数据 |
| | 人工录入/批量导入 | 线下数据手动录入和文件批量导入 |

### 2.3 设计原则

| 原则 | 说明 |
|------|------|
| **数据为根基** | 平台的起点是数据池，一切应用以数据为基础 |
| **接入最大化兼容** | 弱化协议适配问题，追求最大兼容性，接收纳入各种数据源的数据 |
| **行业化专业呈现** | 不同行业适配完全不同的UI和主题，不能只是简单更换背景和插图 |
| **国际化** | 多语言自适应，地图按国家/地区切换，满足海外市场合规 |
| **智慧化** | AI基于预录指标-行业映射，引导式推荐行业，智能预警与决策辅助 |
| **硬件销售引擎** | 软件为引擎，带动自有研发硬件产品销售+拓展业务销售拓品+业务承接能力 |

## 5. 核心使用场景

### 场景1：首次使用 — AI引导

> **流程**：新用户注册 → 选择关注的指标（如：土壤湿度、pH、温度） → AI基于指标-行业映射判断为"智慧种植" → 推荐种植场景界面 → 用户确认或自定义选择行业 → 自动切换对应UI主题和功能模块 → 进入专属大屏/APP首页

### 场景2：智慧种植 — 日常监控与控制

> **流程**：农场主打开APP → 查看今日各传感器数据 → 发现土壤湿度过低 → 收到推送预警 → 一键启动灌溉 → WE-W280控制箱执行继电器输出 → 灌溉完成自动停止

### 场景3：智慧牧业 — 牲畜追踪

> **流程**：牧场主查看GPS地图 → 发现一头牛离开虚拟围栏 → 收到振动+语音报警通知 → 通过APP远程触发项圈语音驱赶

### 场景4：智慧水产 — DO紧急报警

> **流程**：凌晨DO骤降至危险值 → 系统自动电话呼叫养殖户 → 养殖户未接听 → 5分钟后升级通知合作社管理员 → 自动启动增氧机 → DO回升至安全值 → 报警解除

### 场景5：数字护照 — 农产品溯源

> **流程**：出口批次木薯 → 系统自动汇总从种植到采收全链路数据 → 生成数字护照二维码 → 物流环节临时授权录入运输信息（提交后只读） → 买家扫码查看完整溯源信息

### 场景6：家庭庄园 — 草坪/泳池管理

> **流程**：庄园主查看草坪土壤湿度 → 系统提醒该修剪草坪 → 远程启动灌溉 → 切换泳池页面查看水质pH/余氯 → 浊度超标报警 → 一键启动过滤循环

### 场景7：多源数据接入

> **流程**：WEST管理员配置新数据源 → 注册WES-AIOT数据通道 → 同时接入第三方气象API → 配置指标-行业映射 → 数据自动归入指标池 → 按行业归类展示
